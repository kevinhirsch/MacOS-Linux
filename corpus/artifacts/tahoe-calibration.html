macOS Tahoe (26) & iOS 26 — sourced numeric values

Hard values for a Tahoe calibration set, each tagged by provenance (who authored the number) and confidence. Every figure was cross-checked against its primary source; contested numbers were adversarially verified and demoted when the source didn't actually state them.

Compiled 2026-07-03 Method 5-agent fan-out + direct Apple HIG JSON + source verification Scope numeric specs only

Read this first — the load-bearing finding

Apple publishes almost no fixed pt/px numbers for Tahoe. The HIG is overwhelmingly qualitative, and the new design system is deliberately derived, not fixed: corner radii are computed by the concentric rule (radius = parent radius − padding) and vary per window configuration, so for many targets there is no single canonical value to quote.

The real shadow numbers live only inside Apple's Figma/Sketch UI kits as inspectable effect layers. The official "macOS 26" kit (figma.com/@apple, verified, 39.8k users) exists — but its values are readable only by opening the file in an authenticated Figma editor. The community landing page exposes zero inspectable numbers, and I did not sign in. So no Apple-authored shadow value could be extracted here; the shadow rows below are community reconstructions, clearly flagged.

Poison value to purge from any calibration set: the ubiquitous box-shadow: 0 8px 32px rgba(31,38,135,…) tagged "iOS 26" across CSS blogs is recycled 2020 glassmorphism (Hype4/glassmorphism.com default), predating Liquid Glass by ~5 years. Not measured, not Apple. Two independent agents flagged it.

Liquid Glass material — blur & saturation

No true OS measurement exists in any public source. Apple exposes only semantic materials via glassEffect(_:in:) — .regular and .clear — with zero raw parameters. The real effect is a GPU refraction/lensing shader, not a static blur, so it can't be reduced to one blur(Npx). Every number below is a developer's eyeballed reproduction; ranges are wide precisely because they're aesthetic choices.

05

Corner radii & the concentric system

This is where Tahoe breaks the "fixed token" model hardest. Radii are computed, not stored. Apple's own apps ship ~4 different window radii depending on titlebar-only vs toolbar vs sidebar. The one thing Apple states numerically is the rule, via WWDC25 session 356 — the shape formulas below. Everything asking for a single pt radius (sheet, menu, button, field, segmented, window) has no Apple-published value.

To get the real shadow numbers

The only path to Apple-authored shadow / radius values is opening the official kit in an authenticated Figma editor (or the Sketch library locally) and reading the Inspect panel's effect layers for a window, sheet, popover, and card. These require your logged-in session — I did not and cannot sign in.
