Material Lab · strategy

Pixel-Parity Strategy

Synthesized from 30 measured/referenced tokens, five research threads, and a working shader prototype. It answers one question — how far, and by what path, can this Ubuntu box actually reach macOS Tahoe — and maps every measured number to where it gets applied.

The verdict

Every gap is one of three layers, and only the compositor is the real lever. Static appearance (color, radius, font, spacing) is fully closable on your current GNOME with the measured tokens — that's ~80% and it costs nothing. The remaining structural 15–20% (live glass, global menu, squircle window masks) lives in the compositor and only exists on KWin. So the plan is a hybrid: bank the theme-layer win on GNOME now, then trial Plasma alongside (installed, session-switchable, reversible — not a reinstall) to reach for the structural ceiling. No Kubuntu wipe required.

The plan

Three phases, each independently worth doing — you can stop after any one.

~80%
Phase 0 · now

Apply the tokens on GNOME

zero risk · reversible · hours
  • Push the 30 measured tokens into the MacTahoe GTK theme + gsettings
  • Traffic lights 14/24/26pt, window radii, card 9pt, accent palette, selection #2962D9 vs tint #007AFF
  • Menu bar → 31px, blur-my-shell "Tinted" profile for legibility
Delivers: static appearance parity. Ceiling: no live glass / global menu / squircles.
~90%
Phase 1 · trial

Plasma alongside GNOME

reversible · a login away · not a reinstall
  • apt install kde-plasma-desktop; pick Plasma at the login screen, GNOME stays intact
  • MacTahoe-kde + Kvantum + Better Blur (real frosted glass on windows & menus)
  • Global Menu widget, KDE-Rounded-Corners squircle shader, same tokens drive it
Delivers: the structural gaps. Cost: coexistence is messy (dupe apps, DM churn) — commit only if the trial convinces.
~95%
Phase 2 · deep

Write Liquid Glass

only on KWin · days–weeks
  • Fork Better Blur; add the Lab's refraction + specular passes as GLSL
  • The prototype shader ports nearly 1:1 — backdrop sampling is the same shape
  • Tune to tokens: squircle n≈4, restrained refraction, Clear/Tinted states
Delivers: real lensing glass — the best on any Linux DE. Ceiling: still no dynamic-ink vibrancy.

04

Ceilings we accept

Honest — no path on Linux closes these, so the plan doesn't pretend to.

Dynamic ink

Adaptive foreground on glass needs the app's own toolkit to cooperate. Compositor-only can't. ~unreachable.

Squircle exactness

n≈4 approximates it; Apple's real corner is tuned Bézier, published nowhere. Pixel-exact is impossible.

Spring motion

Real physics only inside libadwaita apps you patch; shells do easing curves, not springs.

Refraction fidelity

P2 gets convincing lensing, but not Tahoe's exact specular/edge model — that's a rendering pipeline we can't fully mirror.

Recommended entry point: Phase 0 now — it's the highest value-per-risk, applies everything measured, and improves your live desktop today without touching the DE. Phase 1 is the real decision point: try Plasma in a parallel session, and only commit if the live glass + global menu earn the coexistence mess. Phase 2 is a genuine build project, worthwhile only once you're on KWin. Every number here traces back to a measurement or an Apple reference in the calibration sheet.
